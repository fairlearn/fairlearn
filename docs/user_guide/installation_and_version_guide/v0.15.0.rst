v0.15.0
=======

.. note::

   v0.15.0 is not yet released. This page reflects changes on the current
   :code:`main` branch that will eventually be a part of v0.15.0.

Bug fixes
---------

* Fixed :code:`fetch_diabetes_hospital` so that :code:`as_frame=False` returns numpy arrays instead of raising. The data is now always fetched as a pandas DataFrame internally and converted to numpy when needed. The skipped tests and the obsolete :code:`test_fetch_diabetes_hospital_as_ndarray_raises_value_error` test were removed: :pr:`1636` by :user:`Richard Ogundele <richardogundele>`.

Other improvements
------------------

* Added support for Python 3.14: :pr:`1649` by :user:`Roman Lutz <romanlutz>`.
* Raised the minimum supported :code:`scikit-learn` version to :code:`1.6.0` and removed the :code:`fairlearn.utils._fixes` compatibility shim, which existed to bridge the :code:`scikit-learn` 1.5 ↔ 1.6 API change (:code:`validate_data` import location, :code:`force_all_finite` → :code:`ensure_all_finite` rename, and the wrapped :code:`parametrize_with_checks`). Internal callers now import directly from :code:`sklearn.utils.validation` and :code:`sklearn.utils.estimator_checks`. Also marked the :code:`pytest-mpl` image-comparison tests in :code:`test/unit/postprocessing/test_plots.py` as :code:`xfail(strict=False)` because the committed baselines under :code:`test/unit/plot_snapshots/` are stale for the current matplotlib stack on every platform; they should be regenerated on each target OS as a follow-up: :pr:`1654` by :user:`Roman Lutz <romanlutz>`.
